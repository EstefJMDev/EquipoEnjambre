# FlowWeaver — Phase Gates

## Regla general

No se avanza de fase por completitud cosmética, sino por validación suficiente de la hipótesis correspondiente.

## Gate de salida de Fase 0a

### Condiciones mínimas
- el workspace se entiende
- la agrupación resulta comprensible
- el contenedor genera interés
- el equipo distingue claramente 0a de 0b

### Condiciones de no-paso
- el workspace no se entiende
- la agrupación no genera valor percibido
- se está interpretando 0a como validación de PMF
- el equipo no distingue 0a de 0b

## Gate de salida de Fase 0b

### Condiciones mínimas
- Share Extension funcional
- Episode Detector dual-mode operativo
- sync con ACK e idempotencia funcional
- fallback QR definido
- Privacy Dashboard mínimo funcional
- demo end-to-end del momento mágico

### Condiciones de no-paso
- iCloud sync no es fiable y no existe fallback aceptable
- el usuario no percibe el wow moment
- el sistema parece vigilancia
- broad mode domina sin control
- la experiencia depende de LLM local

## Regla para este repositorio

Los gates se documentan y gobiernan.
No se implementan aquí como lógica de producto.
