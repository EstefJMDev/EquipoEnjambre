# FlowWeaver — Product Thesis

La mayoría de herramientas de automatización obligan al usuario a definir flujos explícitos.
Eso funciona para usuarios expertos, pero deja fuera la mayor parte de las fricciones cotidianas.

FlowWeaver parte de una tesis distinta:

> Las personas no necesitan más herramientas para diseñar automatizaciones manuales.
> Necesitan un sistema que detecte transiciones repetidas de intención y prepare el siguiente paso con la menor fricción posible.

## Qué significa esto en FlowWeaver

En una primera etapa, el sistema no “aprende” todavía hábitos largos.
Primero detecta episodios puntuales de intención real:

* varios recursos guardados
* tema similar
* ventana temporal corta
* contexto suficiente para inferir el siguiente paso

Después, en fases posteriores, el sistema puede aprender patrones longitudinales y gestionar confianza progresiva.

## Qué NO significa

* no es un orquestador de reglas tipo Zapier
* no es un asistente conversacional que espera instrucciones explícitas
* no es vigilancia continua
* no es scraping persistente del contenido completo del usuario

## Valor diferencial

* entiende mejor la continuidad del trabajo
* conecta intención entre dispositivos
* reduce fricción de cambio de contexto
* protege la privacidad de forma verificable
