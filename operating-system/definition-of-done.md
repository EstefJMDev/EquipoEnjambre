# FlowWeaver - Definition Of Done

## Propósito

Definir cuándo un entregable del sistema multiagente puede considerarse cerrado.

## Un entregable está hecho solo si

* respeta roadmap y decisiones cerradas
* pertenece a la fase correcta
* no adelanta trabajo futuro sin aprobación
* no introduce implementación funcional del producto
* deja trazabilidad documental suficiente
* identifica si requiere revisión, handoff o escalado
* usa una plantilla operativa cuando aplica
* es accionable por el siguiente agente sin reinterpretar el contexto

## Señales de incompletitud

* faltan restricciones explícitas
* faltan exclusiones o límites
* faltan owners o siguiente agente
* el texto deja ambigüedad sobre qué valida y qué no valida
* el output depende de una conversación no reflejada en el repo

## Regla de QA

QA Auditor puede reabrir cualquier entregable que no cumpla esta definición,
aunque el contenido esté bien redactado.
